### 2. 新增特性

#### ✨ Note 样式内容区
- 使用飞书的 `note` 标签突出显示主要内容
- 提供更好的视觉层次感

#### 🎯 字段图标系统
每个字段都配有对应的图标:
- 🎯 会话ID
- ⏰/⏱️ 时间相关
- 📋/📄 信息字段
- 🏷️ 标签/类型
- 📥 输入参数
- 📤 输出结果
- 📈 统计信息
- ⚠️ 错误信息
- 💡 原因说明
- 👤 用户信息
- 📁 文件路径
- 🔀 Git状态

#### 🔦 智能高亮
自动高亮重要信息:
- ❌ 失败状态 → 红色高亮
- ✅ 成功状态 → 绿色高亮
- ⚠️ 错误信息 → 总是高亮
- 🔐 权限请求 → 黄色高亮
- 📁 非Git仓库 → 橙色提示

#### 🧰 工具图标映射
不同工具显示不同图标:
- 💻 Bash - 终端命令
- 📖 Read - 文件读取
- ✍️ Write - 文件写入
- ✏️ Edit - 文件编辑
- 🔍 Grep - 内容搜索
- 🗂️ Glob - 文件查找
- 🤖 Task - 子任务
- 🌐 WebFetch - 网络请求

#### 🎨 颜色文本支持
- 时间戳使用灰色文本
- 成功信息使用绿色高亮
- 失败信息使用红色高亮

## 🔧 技术实现

### 代码结构优化

#### 1. 增强的 `_build_card` 方法
```python
# 支持 note 样式
elements.append({
    "tag": "note",
    "elements": [...]
})

# 支持字段图标
field = {
    "name": "字段名",
    "value": "字段值",
    "icon": "🎯",        # 新增
    "highlight": True    # 新增
}

# 支持颜色文本
content = f"<font color='grey'>{text}</font>"
content = f"<font color='red'>{text}</font>"
content = f"<font color='green'>✓ {text}</font>"
```

#### 2. 统一的图标系统
```python
# 通用字段图标
"🎯"  # 会话ID
"⏰"  # 时间
"📋"  # 信息
"📊"  # 状态/统计

# 工具特定图标
def _get_tool_icon(tool_name: str) -> str:
    tool_icons = {
        "Bash": "💻",
        "Read": "📖",
        "Write": "✍️",
        "Edit": "✏️",
        # ...
    }
    return tool_icons.get(tool_name, "🔧")
```

#### 3. 智能高亮逻辑
```python
# 失败时高亮
"highlight": not is_success

# 错误信息总是高亮
"highlight": stop_type == "error"

# 权限请求高亮
"highlight": notification_type == "permission_prompt"
```

## 💡 设计原则

### 1. 信息优先级
```
高 → 中 → 低
────────────
高亮 → 图标+粗体 → 灰色
Note → 普通文本 → 时间戳
```

### 2. 颜色语义
```
🔴 红色 → 错误/失败
🟢 绿色 → 成功/完成
🟡 黄色 → 警告/中断
⚪ 灰色 → 次要信息
```

### 3. 图标选择
- ✅ 直观易懂
- ✅ 与内容相关
- ✅ 统一风格
- ✅ 常见emoji
