# Ship - 开发交付闭环

接收开发需求，自主完成"计划→拆解→编码→数据库→编译→推送→交付"全链路闭环。

## 参数

用户输入格式：自然语言描述开发需求

示例：
- `/ship 活动创建时统一将状态设为Planned`
- `/ship 新增一个按活动类型查询的接口`
- `/ship 修复导入时channelGroup校验报错的问题`

## 执行流程

### 第一步：需求分析与计划

#### 1.1 代码探索

1. 分析用户输入，定位相关代码文件
2. 启动 Explore Agent（最多3个并行）搜索代码库：
   - 1个 Agent：需求明确、涉及已知文件时
   - 多个 Agent：范围不确定、涉及多个模块时（每个 Agent 一个搜索方向）
3. 搜索现有函数、工具类和代码模式，优先复用而非新建
4. 输出探索结果摘要

#### 1.2 需求澄清

基于代码探索结果，梳理以下内容：

1. **疑问点** — 需求描述中不明确的地方
2. **不完善处** — 需求未覆盖但实现时必须决定的细节（如边界条件、错误处理、数据迁移）
3. **冲突点** — 需求与现有代码逻辑或架构的矛盾
4. **假设列表** — 如果无疑问，列出所做的关键假设

**处理规则**：
- 有疑问/冲突 → **逐条列出，等待用户逐一确认后再继续**
- 无疑问 → 列出假设，直接进入下一步

#### 1.3 方案设计

1. **需求重述** — 用清晰的语言重新描述确认后的需求
2. **实施方案** — 分阶段的具体步骤
3. **涉及文件** — 列出需要修改/新增的文件路径
4. **复用清单** — 已有的可复用函数和工具类（含路径）
5. **风险评估** — 潜在风险及应对策略
6. **验证方式** — 如何测试变更的正确性

#### 1.4 创建实施计划

**在编写任何代码之前，必须使用 `/plan` 模式创建全面的实施计划。**

进入 plan 模式后，将方案设计的结果写入 plan 文件，格式如下：

```
## Context
为什么要做这个变更，解决什么问题

## 需求确认
澄清后的最终需求要点

## 实施步骤
分阶段的具体改动（含文件路径）

## 复用清单
已有的可复用代码（含路径）

## 风险
潜在风险及应对

## 验证
如何验证变更正确性
```

#### 1.5 请求确认

**等待用户批准计划后才能开始编码。**

用户可以回复：
- 确认 → 退出 plan 模式，进入第二步
- 修改 → 根据反馈调整计划
- 拒绝 → 重新分析

### 第二步：任务拆解

1. 将计划拆解为可独立执行的子任务
2. 每个子任务明确：修改文件、改动内容、依赖关系
3. 按依赖顺序排列，标注可并行的子任务
4. 逐个子任务执行后续步骤（第三步→第四步→第五步），全部完成后进入第六步

### 第三步：修改代码

1. 按子任务逐个修改文件
2. 遵循项目 CLAUDE.md 中的编码规范
3. 使用 `/claude-dx:java-coding-standards` 确保 Java 编码规范
4. 涉及 REST API 或分层架构时使用 `/claude-dx:springboot-patterns`
5. 涉及实体设计或数据库映射时使用 `/claude-dx:mybatis-plus-patterns`
6. 修改完成后输出变更摘要

### 第四步：执行数据库操作（如有）

1. 如果需求涉及表结构变更（DDL），执行对应 SQL
2. 如果需要初始化数据或更新配置数据（DML），执行对应 SQL
3. 涉及实体与表映射时参考 `/claude-dx:mybatis-plus-patterns`
4. 无数据库操作则跳过此步
5. 数据库连接信息见项目 CLAUDE.md

### 第五步：编译验证

根据项目类型执行编译命令，确认无错误：

- 编译成功 → 继续下一步
- 编译失败 → 分析错误，修复后重新编译，直到通过

### 第六步：推送代码

- 编译通过后，使用 `/claude-dx:git-push` 将代码推送到远程分支

### 第七步：交付确认

输出交付报告：

```
== 交付报告 ==
需求：{需求描述}
子任务：{子任务列表及完成状态}
修改文件：{文件列表}
编译：通过
推送：已推送至远程分支
状态：{通过/失败}
```

- 编译或推送失败 → 分析原因，回到第三步修复，重新走流程

## 注意事项

- 第一步计划必须等用户确认，其余步骤自动执行
- 不要跳过任何步骤，即使改动很小也要完整走完流程

## 插件使用速查

| 插件 | 触发时机 |
|------|---------|
| `claude-dx:java-coding-standards` | 第三步：编写 Java 代码时 |
| `claude-dx:springboot-patterns` | 第三步：涉及 REST API、分层架构时 |
| `claude-dx:mybatis-plus-patterns` | 第三步/第四步：涉及实体设计、数据库映射时 |
| `claude-dx:git-push` | 第六步：推送代码到远程分支 |
